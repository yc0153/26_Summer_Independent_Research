# 4차 시도 정리

## 1. 4차 시도의 목표

3차 시도는 `target_image.jpg`에서 개구부의 위치를 미리 관찰한 뒤, 그 위치를 `AUTO_OPENING_TARGET_FRACTIONS`에 저장해 자동 사각형 탐지에 사용했다. 이 방식은 한 장의 사진에서는 잘 동작하지만, 사진의 촬영 위치나 개구부의 크기가 바뀌면 같은 기준이 맞지 않을 수 있다.

4차 시도에서는 특정 사진의 꼭짓점 좌표, 목표 사각형, 정답 개수를 코드에서 제거했다. 대신 다음과 같은 일반적인 영상처리 규칙으로 후보를 평가한다.

- 이미지 크기에 비례한 Hough 선 길이
- 화면 테두리 선 제거
- 중복 Hough 선 제거
- 이미지 중앙을 기준으로 한 넓은 상·하·좌·우 후보 영역
- 네 직선이 만드는 사각형의 볼록성, 면적, 변 길이
- 네 변이 실제 edge를 따라가는 정도
- 사각형 내부에 포함된 직선 구조

따라서 4차 시도는 특정 이미지의 정답을 외운 모델이 아니라, 여러 사진에 적용해 볼 수 있는 일반화 방향의 프로토타입이다.

## 2. 3차 시도와의 차이

### 3차 시도

3차에서는 다음과 같이 특정 사진에 맞춘 기준이 있었다.

```python
AUTO_OPENING_TARGET_FRACTIONS = ...
```

Hough 직선으로 만든 후보 사각형 중에서 위 기준 꼭짓점과 가장 가까운 후보를 선택했다. 또한 선의 각도와 위치도 현재 사진의 픽셀 범위에 맞춰 제한되어 있었다.

### 4차 시도

4차에서는 목표 꼭짓점과의 거리를 계산하지 않는다.

1. Canny와 HoughLinesP로 직선 후보를 찾는다.
2. 입력 이미지 크기에 비례해 너무 짧은 선을 제거한다.
3. 이미지 가장자리에 붙은 프레임 선을 제거한다.
4. 각도와 직선까지의 거리가 거의 같은 중복 후보를 하나로 줄인다.
5. 상·하 후보는 이미지의 위·아래 영역, 좌·우 후보는 이미지의 왼쪽·오른쪽 영역에서 넓게 선택한다.
6. 네 직선의 교차점으로 사각형 후보를 만든다.
7. 볼록 사각형인지, 면적이 지나치게 작거나 큰지, 네 변이 충분히 긴지 확인한다.
8. 후보 변이 Canny edge를 얼마나 잘 따라가는지 계산한다.
9. 사각형 내부에 직선 구조가 얼마나 포함되는지 계산한다.
10. 위 점수를 합산해 가장 자연스러운 후보를 선택한다.

## 3. 주요 함수

### `deduplicate_hough_segments`

같은 실제 선에서 여러 개의 Hough segment가 검출되는 문제를 줄인다. 선의 방향과 위치가 거의 같으면 길이가 가장 긴 후보 하나만 남긴다.

### `build_hough_pools`

픽셀 좌표를 직접 지정하지 않고, 이미지 너비와 높이에 대한 상대 위치로 후보를 나눈다. 예를 들어 상단 후보는 이미지 높이의 60%보다 위에 있는 선, 하단 후보는 40%보다 아래에 있는 선을 넓게 고려한다.

### `quad_geometry`

네 교차점이 볼록한 사각형을 이루는지 확인하고, 이미지 면적 대비 크기와 최소 변 길이를 검사한다.

### `quad_edge_support`

후보 사각형의 네 변을 따라가며 Canny edge가 실제로 존재하는 비율을 계산한다. 네 변 중 하나만 잘 맞고 나머지가 빈 공간인 후보를 낮게 평가하는 데 사용한다.

### `quad_interior_line_score`

후보 사각형 내부에 포함된 Hough 선의 길이를 합산한다. 철근망처럼 내부에 반복적인 직선 구조가 있는 후보를 우선하기 위한 보조 점수다.

### `score_opening_quad`

edge 지지도, 사각형 면적, 내부 직선 구조를 합산한다. 이 함수에는 특정 사진의 목표 좌표나 정답 카운트가 들어 있지 않다.

## 4. 카운팅 단계

사각형 탐지 뒤의 카운팅 흐름은 3차와 같은 구조를 유지한다.

```text
선택된 사각형
  → perspective warp
  → 검은 영역 제거
  → Hough로 철근 방향 추정
  → affine 축 보정
  → Sobel gradient profile
  → peak 검출
  → 두 방향 개수 출력
```

카운트 숫자 자체는 코드에 `6`, `12`처럼 정답으로 저장되어 있지 않다. 최종 숫자는 profile에서 검출된 peak의 개수로 계산된다. 다만 `peak-percentile`과 최소 거리 값은 여전히 영상처리 휴리스틱이므로, 다양한 사진에서 별도 검증이 필요하다.

## 5. 기존 원본으로 회귀 테스트한 결과

현재 저장소에 원본 사진 자체는 없기 때문에, 3차 시도가 보관해 둔 `00_loaded_input.png`를 입력으로 사용했다.

4차 자동 탐지 결과:

- 탐지 방법: Hough
- 선택된 외곽 사각형: 원래 개구부와 비슷한 위치
- x 방향 카운트: 6개
- y 방향 카운트: 11개

3차의 결과가 x 6개, y 12개였던 것과 비교하면, 외곽 사각형을 목표 좌표로 고정하지 않았기 때문에 경계와 peak 결과가 달라졌다. 이 차이는 4차가 특정 사진의 12라는 결과를 강제로 재현하지 않는다는 점을 보여 준다.

## 6. 현재 한계

4차도 완전히 일반화된 모델은 아니다.

- 사각형 후보를 만들기 위해 이미지의 상·하·좌·우라는 구조적 가정을 사용한다.
- 벽, 거푸집, 이미지 테두리가 철근망보다 강한 edge를 만들면 잘못된 사각형이 선택될 수 있다.
- 철근망이 화면의 아주 한쪽에 있거나 일부만 보이면 상대 위치 조건이 부족할 수 있다.
- profile의 threshold와 최소 거리는 해상도와 철근 간격에 영향을 받는다.

따라서 4차는 3차보다 과적합을 줄인 버전이지만, 일반화가 입증된 상태는 아니다.

## 7. 과적합 검증 방법

다음 사진들을 평가용으로 분리해 두고, 그 사진들의 정답을 보지 않은 상태에서 실행해야 한다.

1. 촬영 위치가 다른 사진 5장 이상 준비
2. 사진마다 실제 x/y 철근 개수를 별도 기록
3. 코드의 파라미터를 평가 사진에 맞춰 다시 조정하지 않기
4. 자동 사각형 탐지 성공률과 x/y counting accuracy를 따로 계산
5. 실패한 사진도 output과 JSON을 함께 저장해 어느 단계에서 실패했는지 확인

이 검증을 통과해야 4차 알고리즘이 한 장의 사진에만 맞춘 것이 아닌지 판단할 수 있다.
