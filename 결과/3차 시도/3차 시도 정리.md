# 3차 시도 정리

2차 시도에서 남아 있던 문제는, 같은 철근이 여러 선분으로 쪼개져 보이거나 가장자리에서 두께가 따로 잡혀서 개수가 흔들린다는 점이었다.  
그래서 3차 시도에서는 `선분 자체를 1차원 위치로 클러스터링`해서 같은 철근을 먼저 묶고, 그 다음에 개수를 세는 방식으로 바꿨다.

## 이번 시도에서 바꾼 점

1. `warp + affine rectification` 흐름은 유지했다.

   - 사진 전체의 원근을 먼저 펴서, 철근이 가급적 수평/수직에 가까워지도록 했다.

2. rectified 이미지에서 `Canny + HoughLinesP`로 선분을 다시 추출했다.

   - 개별 선분의 길이와 각도를 기준으로 수평 후보 / 수직 후보를 나눴다.

3. 같은 철근으로 보이는 선분들을 `DBSCAN`으로 묶었다.

   - 수평 후보는 `y 위치`
   - 수직 후보는 `x 위치`
   - 를 기준으로 1차원 클러스터링했다.

4. 클러스터의 `총 선분 길이`를 support로 써서 약한 잡음을 걸러냈다.

   - 한두 개의 짧은 오검출보다, 길이가 길고 선분이 많이 모인 클러스터를 우선했다.

5. 클러스터 결과가 너무 약하면 기존 `profile peak` 방식으로 fallback 하도록 해 두었다.

   - 완전히 새로운 방법만 쓰는 대신, 실패했을 때 결과가 비지 않도록 안전장치를 넣었다.

## 기대 효과

1. 하나의 철근이 여러 조각으로 검출되어도 하나로 합쳐진다.

2. 철근의 양쪽 edge가 따로 검출되더라도 가까우면 같은 클러스터로 묶일 가능성이 높다.

3. profile peak보다 선분 구조를 더 직접적으로 쓰기 때문에, 끊김이나 부분적인 결손에 조금 더 강하다.

## 저장 위치

- 코드: `결과/3차 시도/rebar_grid_counter.py`
- 결과: `결과/3차 시도/output/target_image/`

## 실행 메모

- 기본 입력 이미지는 `target_image.jpg`를 찾도록 되어 있다.
- 실제 `target_image.jpg`로 다시 실행했고, 결과는 `결과/3차 시도/output/target_image/`에 저장했다.
- 이번 이미지에서는 클러스터 결과가 너무 뭉쳐서 `profile_fallback`이 최종값이 되었다.
- 최종 카운트는 `x축 방향 6개`, `y축 방향 12개`다.
- 합성 격자 테스트에서는 수평 6개, 수직 8개가 안정적으로 묶이는 것도 확인했다.
