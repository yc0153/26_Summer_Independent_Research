# 3차 시도 정리

3차 시도에서는 철근망의 외곽 영역을 지정한 뒤, Hough 선분과 clustering을 이용해 두 방향의 철근 개수를 독립적으로 계산했다.

## 실제 정답

대상 이미지의 실제 철근 개수는 다음과 같다.

- 가로 방향: 6개
- 세로 방향: 12개

## 3차 시도에서 한 일

1. 대상 이미지에서 철근망 외곽 코너를 직접 지정했다.

   - 좌상단 `(104, 154)`
   - 우상단 `(371, 135)`
   - 우하단 `(453, 306)`
   - 좌하단 `(124, 349)`

2. 지정한 네 점으로 원근 보정과 방향 보정을 수행했다.

   - 철근망 외부의 거푸집과 벽 영역은 검출 대상에서 제외했다.

3. 보정된 철근망 이미지에서 Hough 선분을 추출하고 방향별로 clustering했다.

   - 가로 선분은 y 위치 기준으로 1차원 clustering했다.
   - 세로 선분은 x 위치 기준으로 1차원 clustering했다.
   - 하나의 cluster를 하나의 철근으로 간주했다.

4. clustering 결과가 부족하더라도 다른 카운팅 방법으로 전환하지 않았다.

   - 검출 실패 시 해당 결과를 실패로 처리한다.

## Clustering 성능

clustering 결과는 다음과 같다.

- 가로 방향: 6개
- 세로 방향: 10개
- 실제 정답: 가로 6개, 세로 12개

실제 정답과 비교하면 다음과 같다.

- 가로 방향: `6 / 6`, 정확히 검출
- 세로 방향: `10 / 12`, 2개 미검출
- 전체 개수 기준: `16 / 18`, 약 88.9%

가로 방향은 clustering 결과가 정답과 일치했다. 세로 방향에서는 일부 철근의 선분이 충분히 검출되지 않거나 인접 선분과 하나의 cluster로 합쳐져 2개가 부족하게 계산되었다. 따라서 3차 시도의 clustering 방법은 한 방향에서는 안정적이지만, 두 방향 모두를 정확히 계산하지는 못했다.
