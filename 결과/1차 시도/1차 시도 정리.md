# 1차 시도 정리

대상 이미지 `target_image.jpg`에 대해, `새 폴더`에 넣어둔 문서 호모그래피 과제 흐름을 참고해서 먼저 반듯하게 펴고, 그 다음 철근의 x축 방향 / y축 방향 개수를 세는 1차 파이프라인을 만들었다.

## 1차 시도에서 한 일

1. 네 점을 기준으로 perspective warp를 적용했다.

   - 문서 과제처럼 `4개 코너`를 기준으로 이미지를 정면에 가깝게 펼치는 구조를 먼저 만들었다.
   - 마우스로 직접 찍는 방식과 기본 점 세트를 둘 다 염두에 두었다.

2. warp된 이미지에서 철근 방향을 추정했다.

   - CLAHE, Canny, HoughLinesP를 사용해서 주요 직선 방향을 찾았다.
   - 격자가 완전히 수평/수직이 아니기 때문에, 한 번 더 affine rectification을 적용해서 축을 맞췄다.

3. 축이 맞춰진 이미지에서 방향별 피크를 세었다.

   - x축 방향 철근은 행 방향 gradient profile에서,
   - y축 방향 철근은 열 방향 gradient profile에서,
   - 로컬 피크를 찾아 개수를 셌다.

4. 결과를 단계별 이미지로 저장했다.

   - `01_selected_corners.png`
   - `02_perspective_warp.png`
   - `03_hough_edges.png`
   - `04_axis_rectified.png`
   - `05_profiles.png`
   - `06_count_overlay.png`
   - `07_result.json`

현재 1차 시도 결과는 `x축 방향 6개`, `y축 방향 11개`로 저장되어 있다.

## contour가 잘 안 맞은 이유

1. 이 대상은 하나의 닫힌 물체가 아니다.

   - 철근망은 큰 외곽선 하나로 닫힌 덩어리가 아니라, 반복되는 선분들의 집합이다.
   - contour는 닫힌 외곽 경계를 찾는 데 강하지만, 이런 격자 구조는 내부 빈 공간이 너무 많아서 쉽게 끊어진다.

2. 배경과 구조물이 contour를 더 강하게 만든다.

   - 거푸집 판, 기둥, 모서리, 흰 받침, 그림자, 바닥 질감이 threshold나 Canny에서 철근보다 더 크게 잡히는 경우가 많다.
   - 그래서 contour가 철근 하나하나보다 바깥틀이나 잡음 덩어리를 먼저 따라간다.

3. 원근과 흔들림 때문에 선이 잘린다.

   - 기울어진 시점에서는 철근이 평행하게 보여도 실제 픽셀에서는 조금씩 끊기고 겹친다.
   - 이 상태에서 contour를 찾으면 개별 철근이 아니라 조각난 blob들이 생긴다.

4. 밝기 편차가 심하다.

   - 같은 철근도 밝은 부분과 어두운 부분이 섞여서 한 번의 threshold로 안정적으로 분리되지 않는다.
   - contour는 이런 변화에 매우 민감하다.

## 개선 방향

1. contour는 “전체 ROI를 잡는 용도”로만 쓰고, 실제 개수 세기는 선 기반으로 바꾼다.

   - 철근 개수는 contour보다 방향성 선 검출, projection profile, peak clustering 쪽이 더 잘 맞는다.

2. warp 이후에 검은 가장자리와 거푸집을 더 강하게 제거한다.

   - 내부 유효 영역만 남기면 contour나 Hough가 바깥틀에 덜 흔들린다.

3. 수동 코너 보정을 옵션으로 유지한다.

   - 사진마다 원근이 조금씩 달라서 자동값 하나로 고정하면 흔들린다.
   - 필요하면 네 점을 다시 찍을 수 있게 둔다.

4. 2차 시도에서는 contour를 보조 신호로만 쓰고, 카운팅은 line cluster 중심으로 간다.

   - 각 방향의 bar 위치를 좌표축 기준으로 군집화해서 세는 방식이 더 안정적이다.
   - 결과 이미지와 로그도 시도별 폴더에 함께 저장한다.
